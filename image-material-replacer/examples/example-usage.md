## 示例 1：电商产品图生成

### 场景描述
某服装品牌需要为春季新品生成统一风格的产品展示图，保持品牌调性。

### 操作步骤

#### 第1步：上传示例图片
**示例图片**：冬季产品营销图
- 风格：简约现代
- 色调：冷色系（深蓝+白）
- 构图：产品居中，极简背景

**AI 分析结果**：
```
提示词分析：
- 整体风格：极简主义，现代商业风格
- 色彩方案：冷色调为主，高对比度
- 光线：柔和顶光，突出产品质感
- 构图：三分法则，产品位于画面中心
- 背景：纯色或渐变，无干扰元素
- 材质表现：细腻的布料质感，光影过渡自然
```

#### 第2步：输入新文案
**新标题**：
```
春季新品上市
```

**描述文案**：
```
温暖明亮的春季产品展示，采用简约现代的商业风格。
使用柔和的自然光线，营造清新氛围。
色彩方案转为暖色调：浅粉色、淡绿色、米白色。
产品居中展示，细腻的布料质感清晰可见。
背景采用渐变设计，从浅粉到淡黄，象征春季温暖。
画面周围点缀轻盈的植物叶片元素，增添生机。
整体保持极简风格，突出产品本身。
```

####第3步：上传素材图片
**素材图片**：春季新款连衣裙产品照（白底）
- 款式：轻盈飘逸
- 颜色：浅粉色
- 材质：薄纱面料

**AI 素材分析**：
```
素材特征：
- 主要元素：女装连衣裙，飘逸设计
- 主色调：粉色系，温柔浪漫
- 材质感：轻薄透气，适合春夏
- 设计细节：蕾丝边缘，层叠裙摆
```

#### 第4步：生成新图片
**最终提示词**（AI 自动组合）：
```
标题: 春季新品上市

描述: 温暖明亮的春季产品展示，采用简约现代的商业风格。
使用柔和的自然光线，营造清新氛围。
色彩方案转为暖色调：浅粉色、淡绿色、米白色。
产品居中展示，细腻的布料质感清晰可见。
背景采用渐变设计，从浅粉到淡黄，象征春季温暖。
画面周围点缀轻盈的植物叶片元素，增添生机。
整体保持极简风格，突出产品本身。

风格参考: 极简主义，现代商业风格，冷色调为主，高对比度，
柔和顶光，三分法则构图，纯色背景

素材元素: 女装连衣裙，飘逸设计，粉色系，温柔浪漫，
轻薄透气材质，蕾丝边缘，层叠裙摆

要求:
- 保持原图的整体风格和氛围
- 融合新素材图片中的关键元素
- 画面构图合理，16:9横向布局
- 高质量、细节丰富
```

**生成结果**：
- 尺寸：3840×2160 (16:9, 4K)
- 风格：保持简约现代，融入春季元素
- 效果：产品清晰展示，背景温暖渐变，轻盈叶片点缀

## 示例 2：社交媒体配图

### 场景描述
运营团队需要为品牌账号生成一系列风格统一的社交媒体配图。

### 操作步骤

#### 第1步：示例图片分析
**示例图片**：之前高互动的社媒配图
```
AI 分析：
- 风格：扁平化插画风格
- 色彩：明快鲜艳，高饱和度
- 元素：几何图形+简单图标
- 排版：对称布局，标题居中
```

####第2步：新文案
**标题**：
```
新功能发布
```

**描述**：
```

使用几何图形和简约图标构成画面。
色彩方案：蓝色（#2E86DE）为主色，橙色（#FF6B6B）为点缀色。
居中对称的排版，标题"新功能发布"位于画面中上方。
下方展示3个核心功能图标，均匀分布。
背景采用渐变网格，营造科技感。
整体风格年轻活力，适合社交媒体传播。
```

#### 第3步：素材图片
上传产品功能截图或图标素材

####第4步：生成
输出：16:9 社交媒体横图，适合 Twitter/微博等平台

## 高级技巧

### 技巧 1：提示词优化
在描述文案中添加更多细节可以提高生成质量：
- ✅ **具体**："柔和的清晨阳光，45度角侧光"
- ❌ **模糊**："好看的光线"

### 技巧 2：素材预处理
- 去除素材图片的背景（使用抠图工具）
- 调整素材图片的大小和比例
- 确保素材主体清晰

### 技巧 3：多次生成
如果第一次生成不满意：
1. 保持步骤 1-3 不变
2. 优化描述文案
3. 重新执行步骤 4

### 技巧 4：风格混搭
可以在描述中混合多种风格：
```
基础风格：简约现代（来自示例图）
叠加风格：赛博朋克氛围 + 日系配色
```

## 常见问题

### Q: 如何保证多张图片风格一致？
**A**: 使用同一张示例图片，只修改文案和素材，这样能最大程度保持风格统一。

### Q: 素材图片质量要求？
**A**: 建议使用：
- 分辨率 > 1000px
- 文件大小 > 1MB
- 格式：JPG 或 PNG
- 背景干净或已抠图

### Q: 生成时间过长？
**A**:
- 降低图片质量（4K → 2K）
- 简化描述文案
- 避免高峰时段

### Q: 如何保存生成历史？
**A**: 建议在本地建立文件夹管理：
```
project/
├── prompts/        # 提示词文本
├── examples/       # 示例图片
├── materials/      # 素材图片
└── outputs/        # 生成结果
    ├── version-1/
    ├── version-2/
    └── final/
```
